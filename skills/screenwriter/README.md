# Screenwriter Skill — быстрый старт

Универсальный скилл-инструмент для сценариста. Не привязан к конкретной истории. Работает в Claude / Cowork / Claude Code как папка-навык.

---

## ЧТО ВНУТРИ

- **`SKILL.md`** — главное описание скилла (читается первым).
- **`methodology.md`** — Макки + Кэмпбелл + Аристотель.
- **`style-rules.md`** — правила письма Hollywood-формата.
- **`workflow.md`** — режим работы с пользователем.
- **`timing-and-cutting.md`** — оценка экранного времени и резка длины.
- **`templates/`** — пустые шаблоны под твою историю.
- **`tools/`** — генераторы .docx (сценарий, билингв, поэпизодник).

---

## КАК НАЧАТЬ

### Шаг 1. Установка

Скопируй папку `screenwriter-skill/` куда удобно: внутри проекта Claude Code, как user-skill в Cowork (`~/.claude/skills/screenwriter/`), или просто рядом с рабочими файлами.

### Шаг 2. Скажи Клоду

> «Загрузи скилл screenwriter и начнём.»

Клод прочитает SKILL.md, методологию, правила письма и workflow.

### Шаг 3. Принеси материал

Один из вариантов:

**А. У тебя уже есть синопсис / поэпизодник / черновик сцен.**
Скинь файлы — Клод прочитает и спросит, с какой точки начать работу.

**Б. У тебя есть только идея.**
Расскажи в одном-двух абзацах. Клод задаст вопросы и сначала поможет дотянуть синопсис, потом — поэпизодник, потом — сцены.

**В. У тебя есть только название и жанр.**
Заполни `templates/synopsis.template.md` и `templates/characters.template.md`. Дальше — итеративно.

### Шаг 4. Работай по сценам

Стандартный цикл:
1. Ты просишь сцену по поэпизоднику.
2. Клод даёт ОДНУ версию + аргумент.
3. Ты говоришь правки.
4. Клод правит точечно.
5. Когда сцена финал — выгружаешь в .docx через `tools/build_screenplay.js`.

## ИНСТРУМЕНТЫ ВЫГРУЗКИ

### Сценарий (Hollywood-формат)
```bash
cp tools/build_screenplay.js my_scene.js
# открой my_scene.js, заполни массив `screenplay` через slug/action/character/dial/trans
NODE_PATH=/usr/local/lib/node_modules_global/lib/node_modules node my_scene.js
# получишь screenplay.docx
```

### Билингв (диалог + перевод)
```bash
cp tools/build_bilingual.js my_bilingual.js
# заполни через ...dialB("Main lang", "Translation")
node my_bilingual.js
# получишь screenplay-bilingual.docx
```

### Поэпизодник
```bash
cp tools/build_treatment.js my_treatment.js
# заполни через scene("Title", "Body", "[опц.] audit-tag")
node my_treatment.js
# получишь treatment.docx
```

## ТИПОВЫЕ ЗАПРОСЫ К СКИЛЛУ

| Запрос | Что делает Клод |
|---|---|
| «Напиши сцену 5» | Читает поэпизодник → пишет одну версию + аргумент |
| «Это не работает» | Спрашивает один узкий бинарный вопрос → новая версия |
| «Сделай билингв» | Использует `tools/build_bilingual.js` |
| «Проведи аудит причинности» | Идёт по поэпизоднику с тегами ⚠ |
| «Сколько минут получится?» | Считает по типам сцен (см. `timing-and-cutting.md`) |
| «Уложи в X минут» | Даёт план реза с конкретными цифрами |
| «Сделай голос персонажа Y отличным от X» | Сравнивает реплики, предлагает изменения |

---

## ТРИ ВЕЩИ, КОТОРЫЕ КЛОД НЕ ДЕЛАЕТ

1. **Не пишет 5 вариантов** — даёт ОДИН + аргумент.
2. **Не «улучшает» соседние реплики** — меняет только то, что просили.
3. **Не описывает эмоции** — только глаголы действия.

Если клод нарушает — скажи: «Один вариант, не пять» или «Меняй только Х».

---

## ПЕРСОНАЛИЗАЦИЯ СКИЛЛА

Если ты пишешь много фильмов в одном жанре — можешь форкнуть этот скилл и добавить:

- **`reference-films.md`** — список референс-фильмов с разбором сцен.
- **`my-style.md`** — твои личные предпочтения по стилю (напр. «не люблю флешбеки», «всегда кончать на тишине»).
- **`recurring-tropes.md`** — твои повторяющиеся приёмы.

Скилл становится твоим, не общим.
